## fable-to-opus (adapt-first)

Before borrowing any work-method rule — from this repo or anywhere — check whether my own
CLAUDE.md / skills / hooks already cover it. Add only the genuine gap, in the right layer
(always-on config vs a skill/guard vs a hook). Don't duplicate; don't bloat. When unsure which
of these I already have, run `/fable-to-opus` to audit before adding anything.

Verified habits worth checking I cover (full packs in the fable-to-opus repo):
1. **Render verification** — view the rendered output before declaring it done (schema/lint/build pass ≠ visually correct).
2. **Investigation by hypotheses** — for non-obvious bugs, name 2+ causes and falsify each before concluding.
3. **Delegation guards** — every hand-off carries return-schema + source-priority + dedup + "never fabricate, leave unverified as unverified".
4. **Input fidelity** — re-extract fuzzy critical numbers/dates/ids at high fidelity before trusting them.
5. **Confirm before irreversible** — confirm assumptions before large/irreversible work; scale to reversibility, not confidence.
6. **Rhythm & briefing** — quiet burst then one consolidated briefing, and close open choices with a recommended default; never go quiet over verification or risk.
